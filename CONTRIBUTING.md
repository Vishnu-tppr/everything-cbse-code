# Contributing to Everything CBSE Code (ECB)

First off, thank you for considering contributing to Everything CBSE Code! It's people like you that make ECB a great tool for students across the country.

## How Can I Contribute?

### 🧪 Adding New Skills
The heart of ECB lies in its `SKILL.md` files. If you have deep knowledge of a specific CBSE chapter or topic:
1. Navigate to the appropriate grade folder (`10th` or `12th`).
2. Create a new directory for the chapter/topic under `skills/`.
3. Add a `SKILL.md` file following the existing template (NCERT-aligned facts, keywords, and common mistakes).

### 🤖 Refining Agents
If you notice an agent (like the Socratic Tutor or Examiner) could be improved:
1. Edit the relevant file in the `agents/` directory.
2. Focus on improving the "System Prompt" or "Behavior Rules" to better align with CBSE marking schemes.

### 📂 Improving the Knowledge Vault
The `Prasanna/` and `Prasanna-12/` directories contain Obsidian notes.
1. Add high-quality diagrams, formula banks, or topper-style answer patterns.
2. Ensure notes are "atomic" (one concept per note) and use proper backlinks.

### 💻 Developing the MCP Server
If you're a developer:
1. Help us optimize the TypeScript MCP server in `mcp-server/`.
2. Improve the indexing engine or add new tools to the server.

## Style Guidelines

- **NCERT First**: All academic content must be verified against the latest NCERT textbooks.
- **CBSE Marking Scheme**: Follow the official CBSE 1, 2, 3, and 5-mark answer structures.
- **Tone**: Keep the AI's tone encouraging, professional, and academic.

## Pull Request Process

1. Fork the repository.
2. Create a feature branch for your contribution.
3. If you added code or skills, ensure they follow the project's formatting standards.
4. Update the relevant README if you added major new components.
5. Submit a Pull Request with a clear description of what you changed and why.

## Questions?

Feel free to open an issue or reach out to the project maintainers!
