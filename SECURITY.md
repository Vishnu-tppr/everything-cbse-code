# Security Policy

## Supported Versions

We provide security updates for the following versions of Everything CBSE Code:

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < Latest| :x:                |

## Reporting a Vulnerability

We take the security of this project seriously. If you find a security vulnerability, please do not open a public issue. Instead, please report it via the following steps:

1. Send an email to the project maintainer (or open a private security advisory on GitHub if enabled).
2. Include a detailed description of the vulnerability and steps to reproduce it.
3. We will acknowledge your report within 48 hours and provide a timeline for a fix.

## What to Report

- Hardcoded secrets or API keys.
- Remote Code Execution (RCE) vulnerabilities in the MCP server.
- Data leaks or privacy concerns.
- Malicious skill or agent definitions.

Thank you for helping keep the ECB-10 & ECB-12 community safe!
